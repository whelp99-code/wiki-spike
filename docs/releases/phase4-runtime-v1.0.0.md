# Phase 4 Runtime v1.0.0

Phase 4 closes the provider-neutral Memory Runtime contract above the signed Phase 3 Core.
It includes deterministic orchestration, intent/temporal planning, context and egress
minimization, generation-aware retrieval, EvidencePack and verification, recall,
decision, clarification, proactive attention metadata, ActionIntent/WriteProposal,
exact model routing, hierarchical cost, scoped cache, degradation, evaluation, and
body-free operations telemetry.

This release does not include real model quality approval, connector credentials,
external side-effect execution, Phase 5 UI, or production distributed persistence.
Disabled/shadow model routes remain disabled until separate canary evidence exists.

The signed checkpoint is `artifacts/checkpoints/g4/phase4-g4-checkpoint.json`.  The
release tag must be the annotated immutable tag `phase4-runtime-v1.0.0` created only
after all Phase 3, Phase 4 preflight, and G4 conformance checks pass on the merged
commit.
